# 最终质量检查表

## 一、硬性检查

输出前确认：

- [ ] 没有改变用户核心观点。
- [ ] 没有新增未经证实的数据、事实、案例或名人言论。
- [ ] 涉及考试、政策、新闻或公共事件时，没有编造具体题目、官方说法、统计数据或事件细节。
- [ ] 没有把正式文本改得过度随意。
- [ ] 没有把所有表达都改成营销号。
- [ ] 没有过度使用网络热梗。
- [ ] 没有保留明显 AI 套话。
- [ ] 没有连续多个抽象名词堆在一起。
- [ ] 如果是口播稿，读起来不卡嘴。
- [ ] 如果有风格样本，已经参考用户本人的表达习惯。

## 二、AI 味自检问题

逐段问自己：

1. 这句话真人会这么说吗？
2. 这句话是不是像英文翻译过来的？
3. 有没有更具体的生活场景可以替代抽象概念？
4. 这个例子是不是中文用户熟悉？
5. 如果涉及考试、政策或新闻，这个具体说法有没有来源？
6. 有没有“这不仅……更是……”这类万能总结？
7. 有没有“赋能、闭环、生态、抓手”这类高频黑话？
8. 这段是不是太完整、太工整，反而不像真人？
9. 口头禅是不是用太多了？

## 三、口播读感检查

适用于口播稿：

- [ ] 开头 3 秒内能让人知道为什么要听。
- [ ] 每段只讲一个重点。
- [ ] 长句已经拆开。
- [ ] 关键观点有停顿或单独成段。
- [ ] 没有连续 3 句以上同样句式。
- [ ] 没有需要读者“回看”的复杂嵌套句。
- [ ] 结尾不像作文总结。

## 四、中文语境检查

- [ ] 案例不是默认外国企业。
- [ ] 比喻不是汉堡、星巴克、沃尔玛式英文语料模板。
- [ ] 如使用中国企业 / 企业家，没有编造具体事实。
- [ ] 如使用考试、政策、新闻或公共事件，没有把推测写成事实。
- [ ] 生活场景符合目标受众。
- [ ] 网络梗只在合适位置少量使用。

## 五、个人风格检查

如果有用户样本，检查：

- [ ] 是否保留了用户常用的解释方式？
- [ ] 是否保留了用户可接受的口头禅？
- [ ] 是否避开用户明确不喜欢的词？
- [ ] 情绪强度是否接近用户？
- [ ] 句子长度是否接近用户？
- [ ] 案例类型是否接近用户？

## 六、评分表

如用户要求评分，可以按 1-5 分给出：

| 维度 | 1 分 | 3 分 | 5 分 |
|---|---|---|---|
| AI 味 | 套话明显 | 少量模板感 | 基本像真人 |
| 口语自然度 | 拗口 | 基本顺 | 很顺嘴 |
| 中文语境 | 外文案例多 | 部分中文化 | 很接地气 |
| 信息清晰度 | 看不懂 | 能理解 | 清楚好懂 |
| 个人风格 | 不像用户 | 有一点像 | 明显贴近 |
| 专业准确性 | 改偏了 | 基本保留 | 准确且自然 |

## 七、最终自问

如果这段话是用户本人坐在你面前讲出来的，会不会奇怪？

如果奇怪，继续改。
